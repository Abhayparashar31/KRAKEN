🔷 Bing

Bing supports a solid Google-like operator subset, plus one genuinely unique operator worth knowing: contains:.

Not supported on Bing: allintext:, allinurl:, allintitle:, inanchor:, related:, cache:, and there's no public date-range text operator.

Filters

FilterWhat it does
intitle:Finds pages with the keyword in the page title
inurl:Searches for URLs containing the specified keyword
site:Restricts results to a specific website or domain
filetype:Limits results to a specific file type
link:Finds pages that link to a specific URL
contains:Finds pages that LINK TO a file of the given type (e.g. contains:pdf) - different from filetype:, which finds pages that ARE that file type

Operators

OperatorWhat it does
"phrase"Exact phrase match
ORMatches either term
-Excludes a term
()Groups operators for combined logic

Sample Dorks

Pages linking to PDFs
site:example.com contains:pdf

Finds pages on a domain that link out to PDF files - useful for locating document indexes/repositories.

Exposed backups
site:example.com inurl:backup OR inurl:.env

Finds indexed backup/config files on a target domain.

Name + document type
intitle:"John Doe" filetype:docx

Finds Word documents with a name in the title.

When to use Bing dorking

Cross-verification — Bing indexes some pages Google misses or de-prioritizes, useful as a second opinion during recon.
Finding document repositories — contains: is genuinely useful for locating pages that aggregate links to a filetype, which Google has no direct equivalent for.
Corporate/government sites — Bing has historically indexed more of the .gov/.mil and enterprise SharePoint space well.
