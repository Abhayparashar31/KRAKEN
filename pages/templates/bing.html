Bing

A field guide to advanced search-operator reconnaissance on Bing

Bing has the richest operator set among the "Google-style" engines, including several — like infrastructure-mapping and anchor-text search — that Google has since dropped or heavily restricted.

Authorized use only
01 / Core Operators

The syntax you'll actually use

Combine several in one query for precision — that's where Bing's extra operators really pay off.

OperatorWhat it doesExample
site:Restricts results to a domainsite:microsoft.com
filetype:Limits results to a file extensionfiletype:xlsx
intitle:Word must be in the page titleintitle:advisory
inbody:Word must appear in the page body specificallyinbody:"internal only"
inanchor:Word appears in inbound link anchor textinanchor:"download now"
ip:Finds other sites hosted on the same IP addressip:192.0.2.10
feed:Finds RSS/Atom feeds matching a termfeed:security advisory
contains:Pages that link to a file of a given typecontains:pdf "quarterly report"
loc: / location:Restricts results to a countryloc:UK "budget announcement"
02 / Illustrative Patterns

Common reconnaissance categories

Categories most public OSINT and infrastructure-mapping guides reference — shown at the pattern level so you understand the intent behind each.

ip:203.0.113.5

Reveals other domains co-hosted on the same server — a useful pivot when mapping your own organization's infrastructure footprint from public data.

contains:pdf site:.gov "budget 2024"

Surfaces government pages that link out to PDF budget documents, even if the PDF itself isn't directly indexed.

inanchor:"login" site:target.com

Finds pages that other sites link to using "login" as anchor text — a different discovery angle than a plain URL search.

inbody:"internal use only" filetype:pdf

Restricts the sensitivity-marker search specifically to body text, filtering out pages that only mention the phrase in a title or caption.

feed:"security advisory" site:target.com

Locates an organization's RSS/Atom feed for security advisories, useful for setting up automated monitoring.

loc:DE site:target.com "datenschutz"

Combines a country restriction with a language-specific term to find region-specific compliance pages.

03 / Pro Tips

Getting good results

01

Use ip: only on your own infrastructure

It's the single most powerful operator in this list for infrastructure mapping — reserve it for auditing systems you're authorized to review.

02

Combine inbody: and intitle: for precision

Splitting title vs. body matching filters out false positives where a keyword only appears incidentally in a heading.

03

Use contains: to find link pages, not just files

It surfaces pages that reference a file type without the file itself being indexed — useful when the original document has been taken down.

04

Set the region with loc: before assuming absence

Bing's default region can hide relevant results — explicitly setting a country often surfaces more.

05

Use Bing Webmaster Tools for backlink data instead of link:

Bing's own webmaster dashboard gives far more reliable backlink data than the legacy search operator.

06

Run dorks against your own org first

The highest-value use of Bing's extra operators is auditing your own domain's exposure and infrastructure footprint.

04 / Scope & Ethics

Where the line is

Read before you search
  • Authorization matters, especially for ip:. Infrastructure-mapping operators are appropriate for auditing your own organization, not for reconnaissance against a third party without permission.
  • Finding ≠ accessing. Discovering an exposed file or login page is different from downloading or logging into anything you find.
  • Report, don't exploit. If you stumble on someone else's real exposed data, the responsible move is good-faith disclosure, not further probing.
  • Respect site terms. Some sites explicitly disallow automated scraping of indexed content even when technically reachable.
