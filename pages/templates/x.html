X (Twitter)

A field guide to advanced search-operator reconnaissance on X (OSINT)

X's advanced search syntax lets you filter its public post history far more precisely than the default search box — by account, date, engagement, media type, and location. Used well, it's a core OSINT and investigative research tool: journalists verifying timelines, researchers tracking public discourse, and analysts monitoring brand or security mentions all lean on it daily.

Authorized use only
01 / Core Operators

The syntax you'll actually use

These work in the X search bar and at x.com/search. Combine several in one query — that's where precision comes from. Most also work through X's advanced search UI if you'd rather build them visually.

02 / Illustrative Patterns

Common investigative queries

Categories most OSINT and digital-verification guides reference — shown at the pattern level so you understand the intent behind each, not as a script to run against private individuals.

Timeline verification

Pinning down when something was first posted

"incident name" since:2024-06-01 until:2024-06-03 filter:media

Narrows a fast-moving story to a tight window with visual evidence, useful for confirming what was known and when.

Brand / security monitoring

Catching mentions before they trend

"yourcompany" ("down" OR "outage" OR "hacked") -filter:replies

Flags early chatter about an outage or breach so a comms or security team can respond before it snowballs.

Account activity audit

Reviewing an org's own public posture

from:yourcompany filter:links min_faves:50

A quick way for a team to review their own most-engaged posts that shared external links, for a content or security review.

Geolocated reporting

Finding on-the-ground posts near an event

"protest" near:"City Name" within:10km filter:media

Surfaces geotagged eyewitness media for verification workflows — cross-check against other sources before treating it as confirmed.

Public discourse trends

Gauging reaction volume over time

"policy name" min_retweets:1000 since:2024-01-01

Filters to only the highest-engagement posts on a topic, useful for sampling the loudest public reactions rather than every mention.

Cross-referencing sources

Finding who first shared a link

url:"example.com/report" -filter:replies

Helps trace the earliest accounts to share a specific URL, useful for source attribution and spotting coordinated amplification.

03 / Field Notes

Tips for getting good results

01

Stack operators instead of running separate searches

Combining from: + since:/until: + a quoted phrase gets you a tight result set in one query.

02

Use the Advanced Search UI to build syntax

x.com/search-advanced lets you set filters visually and copy the generated query string — a good way to learn the syntax by example.

03

Engagement thresholds cut through noise

min_retweets: and min_faves: are the fastest way to skip low-signal posts on a high-volume topic.

04

Archive what you find

Posts and accounts can be deleted or made private. If a post is evidentiary, capture a timestamped screenshot or use a web archive tool at the time you find it.

05

Corroborate geotagged content

Location and media filters surface candidates, not confirmed facts — verify against other independent sources before treating a post as ground truth.

06

Run the same queries against your own org

The highest-value use for most teams is inward-facing: checking what's publicly discoverable about your own brand, employees, or leaked internal terms.

04 / Scope & Ethics

Where the line is

Read before you search

These operators only surface content already public on X — they aren't a way around privacy settings, and finding something doesn't make every use of it appropriate:

  • Public posts, not private lives. A post being publicly searchable doesn't license using it to dox, harass, or build a profile on a private individual.
  • Respect X's Terms and rate limits. Automated scraping of search results at scale falls outside normal use and can violate the platform's ToS.
  • Verify before you amplify. Geotagged or timestamped posts can be misleading, mislabeled, or old — corroborate with independent sources before treating them as confirmed.
  • Report, don't exploit. If a search surfaces someone else's exposed personal data or credentials, the responsible step is reporting it, not further digging.
